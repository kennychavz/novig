## Critical Implementation Notes

### 1. Order Book: HTML table, not Recharts

Recharts is for the mid-price timeline and lag sparkline. The order book is a plain `<table>` with depth bars as background overlays. This is 10x faster to code, renders better at 10Hz, and looks more professional for trading UIs.

### 2. State Accumulation: capped buffer via useRef

WebSocket delivers state at 10Hz, Recharts needs historical data. Use a `useRef` with max 50 entries to avoid memory leaks, copy to `useState` for chart renders.

## Module Responsibilities

### App.tsx
Single component file containing:

**WebSocket hook (inline):**
- `useReplicaSocket(url)` → `{ state, connected, chartData }`
- Auto-reconnect with exponential backoff: 1s, 2s, 4s, max 10s
- Resets buffer on URL change (replica switch)

**Layout (grid):**
- Pitch (center) — SVG 100x50 field with ball position, goal zones
- OrderBook (right) — HTML table with depth bars, 5-level bids/asks
- Mid-price timeline (below pitch) — Recharts ComposedChart
- Scoreboard (top) — score, half, clock (MM:SS)
- Metrics (bottom) — lag sparkline (SVG), replica_id
- Controls (bottom) — replica picker, wake/sleep, chaos slider
- Mini-Console (bottom) — client-side event log with capped array

**Types (inline):**
```typescript
interface WorldState {
  replica_id: string
  lag_ms: number
  game: { x: number; y: number; score: string; half: number; clock: number }
  book: { bids: [number, number][]; asks: [number, number][] }
}
```

## Key Interactions

```
User opens UI → connects WS to replica-1 (default) → receives WorldState at ~10Hz
User clicks "Wake" on replica-2 → POST /api/control {"action": "wake"} to localhost:8002
User switches to replica-2 → WS reconnects, history buffer resets
User enables chaos → POST /api/chaos {"lag_ms": 3000} → replica lags, UI shows it
```
