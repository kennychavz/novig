## Critical Implementation Notes

### 1. Order Book: HTML table, not Recharts

Recharts is for the mid-price timeline and lag sparkline. Do NOT use it for the order book.
A 5-level bid/ask book is a table, not a chart. Use a plain `<table>` or CSS grid with
depth bars as background overlays:

```typescript
// Order book row with depth bar
<tr className="relative">
  <td className="text-right font-mono text-pos">{bid.price.toFixed(2)}</td>
  <td className="text-right font-mono">{bid.size}</td>
  {/* Depth bar: proportional width behind the row */}
  <div
    className="absolute inset-y-0 right-0 bg-pos/20"
    style={{ width: `${(bid.size / maxSize) * 100}%` }}
  />
</tr>
```

This is 10x faster to code, renders better at 10Hz, and looks more professional for trading UIs.

### 2. State Accumulation: capped buffer via useRef

The WebSocket delivers current state at 10Hz, but Recharts needs an array of historical
data for the mid-price timeline and lag sparkline. Use a `useRef` with a max length to
avoid memory leaks:

```typescript
const MAX_HISTORY = 50;
const historyRef = useRef<{ ts: number; mid: number; lag: number }[]>([]);

// Inside WS onmessage:
historyRef.current = [
  ...historyRef.current.slice(-(MAX_HISTORY - 1)),
  { ts: state.game.clock, mid: state.book.bids[0][0], lag: state.lag_ms }
];
setChartData([...historyRef.current]);  // trigger re-render
```

Key: `useRef` holds the accumulator (no re-render on push), `useState` holds the snapshot
that Recharts reads. Only copy to state when you want to re-render the chart (every tick
or throttled).

### Custom Tooltip

```typescript
const CustomTooltip = ({ active, payload, label }) => {
  if (!active || !payload?.length) return null;
  return (
    <div className="bg-card/95 backdrop-blur border border-border/50 rounded-lg p-3 shadow-lg">
      <div className="text-xs text-muted-foreground mb-2">{label}</div>
      <div className="space-y-1">
        {payload.map((entry) => (
          <div key={entry.name} className="flex items-center justify-between gap-4">
            <span className="text-xs font-medium text-foreground">{entry.name}</span>
            <span className="text-xs font-mono text-primary">{entry.value?.toFixed(2)}</span>
          </div>
        ))}
      </div>
    </div>
  );
};
```

## Module Responsibilities

### App.tsx
Single component file. Contains:

**WebSocket hook (inline):**
- `useReplicaSocket(url)` -> `{ state, connected, chartData }`
- Opens WS to selected replica, parses WorldState JSON
- Accumulates history into capped `useRef` buffer (max 50 points)
- Copies buffer to `chartData` state on each tick for Recharts
- Auto-reconnect with exponential backoff on disconnect

**Layout (grid):**
- Pitch (center) — SVG 100x50 field with ball position, goal zones
- OrderBook (right) — HTML table with depth bars, 5-level bids/asks
- Mid-price timeline (below pitch) — Recharts ComposedChart from chartData
- Scoreboard (top) — score, half, clock (MM:SS)
- Metrics (bottom) — lag sparkline (SVG), replica_id, events/sec
- Controls (bottom) — replica picker, wake/sleep, chaos slider

**Types (inline):**
```typescript
interface WorldState {
  replica_id: string
  lag_ms: number
  game: { x: number; y: number; score: string; half: number; clock: number }
  book: {
    bids: [number, number][]
    asks: [number, number][]
  }
}
```

## Key Interactions

```
User opens UI:
  -> connects WS to replica-1 (default)
  -> receives WorldState at ~10Hz
  -> renders pitch + book + metrics
  -> accumulates history into capped buffer for charts

User clicks "Wake" on replica-2:
  -> POST /api/control {"action": "wake"} to localhost:8002
  -> replica-2 starts consuming from Redis

User switches to replica-2:
  -> WS disconnects from replica-1
  -> WS connects to replica-2
  -> history buffer resets (new replica = new timeline)
  -> UI now shows replica-2's view (may have different lag)

User enables chaos on replica-1:
  -> POST /api/chaos {"lag_ms": 3000} to localhost:8001
  -> replica-1 starts lagging
  -> if user switches back, they see stale data + high lag_ms
```

## Logs & Status Panel

### Status Indicator

A glowing status dot rendered next to the Replica ID. Derived from `lag_ms` in the WS payload:

```typescript
type StatusLevel = "HEALTHY" | "LAGGING" | "DISCONNECTED";

function getStatus(lagMs: number, connected: boolean): StatusLevel {
  if (!connected) return "DISCONNECTED";
  if (lagMs < 50) return "HEALTHY";
  if (lagMs < 200) return "LAGGING";
  return "DISCONNECTED";
}

function statusColor(status: StatusLevel): string {
  if (status === "HEALTHY") return "hsl(var(--pos))";       // green
  if (status === "LAGGING") return "hsl(var(--warn))";      // yellow
  return "hsl(var(--neg))";                                  // red
}
```

### Mini-Console Log Box

A small scrollable `<div className="font-mono text-xs">` at the bottom of the UI.
Pushes local client-side events into a capped array — no backend log streaming needed.

```typescript
const MAX_LOG_ENTRIES = 100;
const logsRef = useRef<string[]>([]);

function pushLog(msg: string) {
  const ts = new Date().toLocaleTimeString("en-US", { hour12: false });
  logsRef.current = [
    ...logsRef.current.slice(-(MAX_LOG_ENTRIES - 1)),
    `[${ts}] ${msg}`,
  ];
  setLogs([...logsRef.current]);
}
```

Events to log:
- WS connect/disconnect: `Connected to replica-1`, `Disconnected from replica-1`
- Chaos mode: `Chaos mode activated: 3000ms delay injected`
- Lag warnings: `Warning: Replica falling behind, lag > 2000ms`
- Recovery: `Catching up... lag returning to normal`
- Replica switch: `Replica switched to rep-2`
- Control actions: `Wake command sent to replica-1`, `Sleep command sent to replica-1`

Auto-scrolls to bottom on new entries. Styled with glassmorphism card panel and muted text.
