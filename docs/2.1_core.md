# 2.1 Core Layer

The authoritative brain. Owns all game state, runs the physics sim at 10Hz, computes mid-price and volatility, publishes events to Redis Streams.

## Directory Structure

```
core/
├── main.py             # Entry point — connects Redis, starts game loop, graceful shutdown
├── engine.py           # All game logic: ball physics, pricing, volatility, match state
├── publisher.py        # Redis Streams XADD wrapper + connection retry
├── config.py           # All tunables (tick rate, field dims, vol params, Redis URL)
├── requirements.txt
├── Dockerfile
└── tests/
    └── test_engine.py  # pytest: physics, pricing, volatility, serialization
```

## Module Responsibilities

### main.py
- Entry point: `python main.py`
- Connects to Redis (retry loop on startup)
- Instantiates `Engine` (from engine.py) and `Publisher` (from publisher.py)
- Runs the 10Hz async tick loop:
  ```
  while running:
      event = engine.tick()
      publisher.publish(event)
      sleep until next tick
  ```
- Graceful shutdown on SIGINT/SIGTERM

### engine.py
All game logic lives here — ball physics, pricing, volatility, match lifecycle.

**Ball physics (100x50 field):**
- State: `(x, y, vx, vy)`
- Random walk with momentum: velocity perturbed each tick, position updated
- Wall bounces at y=0, y=50
- Goal detection at x=0 and x=100 (within goal zone y=20..30)
- Goal: reset ball to center (50, 25), update score
- Halftime (clock=2700): swap sides, half=2
- Full time (clock=5400): match ends

**Mid-price:**
- Linear function of ball X: `mid = base_price + (x / 100) * price_range`
- x near team A goal -> low price, x near team B goal -> high price

**Volatility:**
- `vol = base_vol` in midfield
- `vol = base_vol * spike_multiplier` when `x < 15` or `x > 85`
- Smooth transition in zones 15-25 and 75-85

**tick() returns a dict:**
```python
{
    "ts": time.time(),
    "ball_x": 88.5,
    "ball_y": 25.1,
    "vol": 0.45,
    "mid": 69.25,
    "score": "0-0",
    "half": 1,
    "clock": 2340.5
}
```

### publisher.py
- Wraps `redis.xadd("game:events", event_dict)`
- Flattens event dict to strings for Redis hash fields
- Trims stream to `MAXLEN ~1000` to bound memory
- Reconnect with exponential backoff on connection loss

### config.py
```python
# Tick rate
TICK_RATE_HZ = 10

# Field
FIELD_W, FIELD_H = 100, 50
GOAL_Y_MIN, GOAL_Y_MAX = 20, 30

# Pricing
BASE_PRICE = 50.0
PRICE_RANGE = 40.0          # mid ranges 50..90

# Volatility
BASE_VOL = 0.15
VOL_SPIKE_MULTIPLIER = 3.0
GOAL_ZONE = 15              # distance from goal line for vol spike

# Match
MATCH_DURATION = 5400       # 90 min simulated

# Redis
REDIS_URL = "redis://redis:6379"
STREAM_KEY = "game:events"
STREAM_MAXLEN = 1000
```

## Data Contract (Core -> Redis)

Each `XADD` writes these fields to `game:events`:

| Field | Type | Example | Description |
|-------|------|---------|-------------|
| `ts` | float | `1709654321.12` | Unix timestamp |
| `ball_x` | float | `88.5` | Ball X position (0-100) |
| `ball_y` | float | `25.1` | Ball Y position (0-50) |
| `vol` | float | `0.45` | Current volatility |
| `mid` | float | `69.25` | Current mid-price |
| `score` | str | `"0-0"` | Match score |
| `half` | int | `1` | Current half |
| `clock` | float | `2340.5` | Match clock (seconds) |

## Lifecycle

```
startup:
  connect redis (retry loop)
  init ball at center (50, 25)
  init score "0-0", half=1, clock=0

tick loop (10Hz):
  engine.tick()         # advance physics, compute mid + vol
  publisher.publish()   # XADD to game:events
  sleep until next tick

match events:
  goal -> reset ball, update score
  halftime (clock=2700) -> swap sides, half=2
  full time (clock=5400) -> publish final event, stop
```

## Tests (pytest)

Run: `pytest core/tests/ -v`

### test_physics_boundaries
Run `engine.tick()` 1,000 times in a loop. Assert that ball coordinates never
exceed the field: `0 <= x <= 100`, `0 <= y <= 50`.

```python
engine = Engine()
for _ in range(1000):
    event = engine.tick()
    assert 0 <= event["ball_x"] <= 100
    assert 0 <= event["ball_y"] <= 50
```

### test_goal_detection_and_reset
Manually set ball to `x=99, y=25` with positive velocity. Call `tick()`.
Assert score increments (`"0-0"` -> `"1-0"`) and ball resets to `(50, 25)`.

```python
engine = Engine()
engine.ball_x = 99
engine.ball_y = 25
engine.ball_vx = 5.0  # moving toward goal at x=100

event = engine.tick()
assert event["score"] == "1-0"
assert event["ball_x"] == 50
assert event["ball_y"] == 25
```

### test_pricing_extremes
Set ball to x=50, x=0, x=100. Assert mid-price equals expected values.

```python
engine = Engine()

engine.ball_x = 50
event = engine.tick()
assert event["mid"] == config.BASE_PRICE + 0.5 * config.PRICE_RANGE  # midfield

engine.ball_x = 0
event = engine.tick()
assert event["mid"] == config.BASE_PRICE  # minimum

engine.ball_x = 100
event = engine.tick()
assert event["mid"] == config.BASE_PRICE + config.PRICE_RANGE  # maximum
```

### test_volatility_spikes
Set ball to x=50, capture vol. Set ball to x=5. Assert new vol equals
`BASE_VOL * VOL_SPIKE_MULTIPLIER`.

```python
engine = Engine()

engine.ball_x = 50
event_mid = engine.tick()

engine.ball_x = 5
event_danger = engine.tick()

assert event_danger["vol"] == config.BASE_VOL * config.VOL_SPIKE_MULTIPLIER
assert event_mid["vol"] == config.BASE_VOL
```

### test_redis_payload_serialization
Call `tick()`, intercept the output. Assert it's a flat dict with all required keys
and no nested objects.

```python
engine = Engine()
event = engine.tick()

required_keys = {"ts", "ball_x", "ball_y", "vol", "mid", "score", "half", "clock"}
assert required_keys == set(event.keys())

# All values must be flat (str, int, float) — no nested dicts/lists
for v in event.values():
    assert isinstance(v, (str, int, float))
```
