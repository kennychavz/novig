## Step 2: WebSocket Hook + Dashboard Layout ✅ DONE

Build the core hook and the grid layout in `App.tsx`.

**WebSocket hook** (`useReplicaSocket` inline in App.tsx):
- Takes `url: string`, returns `{ state: WorldState | null, connected: boolean, chartData }`
- Opens WS, parses JSON `WorldState` on message
- `useRef` buffer capped at `MAX_HISTORY = 50` for chart data accumulation
- Copies buffer to `chartData` state each tick
- Resets buffer on URL change (replica switch)
- Auto-reconnect with exponential backoff: 1s, 2s, 4s, max 10s

**Types** (inline in App.tsx):
```typescript
interface WorldState {
  replica_id: string
  lag_ms: number
  game: { x: number; y: number; score: string; half: number; clock: number }
  book: { bids: [number, number][]; asks: [number, number][] }
}
```

**Layout** (grid shell, placeholder content):
- CSS grid: pitch center, book right, scoreboard top, metrics + controls bottom
- Wrap each section in glassmorphism panel: `bg-card/80 backdrop-blur-xl rounded-2xl border border-border/30`

## Step 5: Logs & Status Panel ✅ DONE

Add observability directly into the UI — a status indicator and a mini-console log box.

**Status Indicator (Green/Yellow/Red dot):**
- Derive from `lag_ms` already in the WebSocket payload
- `HEALTHY` (green): `lag_ms < 50`
- `LAGGING` (yellow): `lag_ms < 200`
- `DISCONNECTED` (red): `lag_ms >= 200` or WS disconnected
- Render as a glowing dot next to the Replica ID on the dashboard

**Mini-Console Log Box:**
- Scrollable `<div className="font-mono text-xs">` at the bottom of the UI layout
- Client-side event log — push local events into a capped `useRef` array (max ~100 entries)
- Events to log:
  - `[HH:MM:SS] Connected to replica-1`
  - `[HH:MM:SS] Disconnected from replica-1`
  - `[HH:MM:SS] Chaos mode activated: 3000ms delay injected`
  - `[HH:MM:SS] Warning: Replica falling behind, lag > 2000ms`
  - `[HH:MM:SS] Catching up... lag returning to normal`
  - `[HH:MM:SS] Replica switched to rep-2`
  - `[HH:MM:SS] Wake command sent to replica-1`
  - `[HH:MM:SS] Sleep command sent to replica-1`
- Auto-scroll to bottom on new entries
- Styled with the card glassmorphism panel, muted text colors
- No backend log streaming needed — all events are derived from local WS state and user actions

**Commit**: `feat(ui): add status indicator and mini-console log panel`

**Dockerfile** (Nginx serves on port 80 internally, mapped to 3000 in docker-compose):
```dockerfile
FROM node:20-alpine AS build
WORKDIR /app
COPY package*.json .
RUN npm ci
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
EXPOSE 80
```

Note: `EXPOSE 80` not `EXPOSE 3000` — Nginx listens on 80 by default. The host
mapping `3000:80` is handled in `docker-compose.yml`.

Run tests: `npx vitest run`

## Implementation Notes

- **Vite version**: Scaffolded with Vite 7.x (latest at time of implementation), which uses `tsc -b` instead of `tsc` for the build script. All compatible.
- **Lag sparkline**: Implemented as a standalone `LagSparkline` component using raw SVG (not Recharts), matching the spec's SVG pattern with gradient fill and Gaussian blur glow.
- **Events/sec counter**: Not implemented — would require tracking message arrival rate which adds complexity for minimal observability value. Can be added later.
- **`lucide-react`**: Installed per spec but not used in current components. Available for future icon needs.
- **Test count**: 4 tests total (the lag color test has 2 separate `it()` blocks for low and high lag). All pass.
- **Controls `pushLog`**: Added as a required prop to `Controls` so replica switch, wake/sleep, and chaos actions are logged to the mini-console.
