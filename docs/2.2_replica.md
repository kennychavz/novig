# 2.2 Replica Layer

The read-only follower(s). Consumes game events from Redis Streams, computes Avellaneda-Stoikov spreads, generates a 5-level order book, serves state via REST + WebSocket.

## Directory Structure

```
replica/
├── main.py             # FastAPI app: routes, WS endpoint, background consumer loop
├── engine.py           # A-S spread calc, 5-level book generation, local state management
├── publisher.py        # WebSocket broadcast manager (fan-out to connected clients)
├── config.py           # Replica tunables + A-S parameters

├── Dockerfile
└── tests/
    └── test_replica.py # pytest + httpx: A-S model, book validity, API endpoints
```

## Critical Implementation Notes

### 1. Async Redis (required)

Standard `redis.Redis` with `XREAD BLOCK` will freeze the entire FastAPI event loop —
WebSockets drop, REST endpoints time out. Must use the async client:

```python
import redis.asyncio as redis

r = redis.Redis.from_url(config.REDIS_URL, decode_responses=True)
messages = await r.xread({"game:events": last_id}, block=1000)
```

`requirements.txt` must pin `redis >= 5.0.0` for the async module.

### 2. Use `block=1000` not `block=0` (critical)

`block=0` blocks indefinitely until a new message arrives. If the UI sends
`POST /api/control {"action": "sleep"}` while the loop is stuck in `block=0`,
it won't re-evaluate `is_active = False` until a new Redis event pushes through.

Use `block=1000` (1-second timeout). If no message arrives in 1 second, `xread` returns
`None`, the loop continues, checks `is_active`, and can cleanly go back to idle or sleep.

### 3. CORS Middleware (required)

The React UI (port 3000) makes POST requests to replicas (ports 8001/8002). Browsers
block cross-origin requests without CORS headers.

```python
from fastapi.middleware.cors import CORSMiddleware
app.add_middleware(CORSMiddleware, allow_origins=["*"], allow_methods=["*"], allow_headers=["*"])
```

### 4. Modern Lifespan (not deprecated @on_event)

Use FastAPI's lifespan context manager to start/stop the consumer cleanly:

```python
from contextlib import asynccontextmanager
import asyncio
from fastapi import FastAPI

@asynccontextmanager
async def lifespan(app: FastAPI):
    task = asyncio.create_task(consume_redis_stream())
    yield
    task.cancel()

app = FastAPI(lifespan=lifespan)
```

### 5. Wake-up cursor: `last_id = "$"` (not "0")

When `is_active` flips to `True`, reset `last_id = "$"` so the replica only consumes
events published *after* wake-up. Using `"0"` would replay the entire stream history,
flooding the frontend with thousands of stale book updates.

```python
# Inside POST /api/control handler:
if action == "wake":
    last_id = "$"       # skip history, only new events
    is_active = True
```

## Module Responsibilities

### main.py
Everything that faces the network lives here — FastAPI app, routes, WS, consumer loop.

**FastAPI app (lifespan pattern):**
```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    task = asyncio.create_task(consume_redis_stream())
    yield
    task.cancel()

app = FastAPI(lifespan=lifespan)
```

**Routes:**
- `GET /api/state` — current WorldState snapshot (REST fallback)
- `GET /api/health` — health check + metrics (lag, uptime, events consumed)
- `POST /api/control` — `{"action": "wake"}` or `{"action": "sleep"}` to toggle consumer
- `POST /api/chaos` — `{"lag_ms": 5000}` injects artificial delay before WS broadcast

**WebSocket endpoint:**
- `WS /ws` — accepts client connections, delegates to publisher.py for broadcast

**Background consumer (`consume_redis_stream`):**
- Uses `redis.asyncio` client — never blocks the event loop
- Hibernation pattern:
  - Starts with `is_active = False`
  - Inactive: `await asyncio.sleep(1)` (lightweight idle)
  - Activated via `POST /api/control {"action": "wake"}`
  - On wake: `last_id = "$"` (skip history, only new events)
- Active loop:
  1. `await r.xread({"game:events": last_id}, block=1000)`
  2. Parse event fields
  3. `engine.update(event)` — updates local state + computes book
  4. `await publisher.broadcast(world_state)` — fans out to WS clients
  5. Track `lag_ms = now - event.ts`
- Auto-reconnect on Redis connection loss
- Reads `REPLICA_ID` from env (e.g., "rep-1", "rep-2")

### engine.py
All replica-side computation: local state + Avellaneda-Stoikov book generation.

**Local state:**
- Holds replicated game state: ball position, score, half, clock, vol, mid
- `update(event)` — applies incoming event, recomputes book
- `get_world_state()` — returns full snapshot for REST/WS

**Avellaneda-Stoikov model:**

Reservation price:

$r = s - q \cdot \gamma \cdot \sigma^2 (T - t)$

Optimal spread:

$\delta = \gamma \cdot \sigma^2 (T - t) + \frac{2}{\gamma} \ln(1 + \frac{\gamma}{k})$

Bid and ask derived as:

$\text{bid} = r - \frac{\delta}{2}$, $\text{ask} = r + \frac{\delta}{2}$

Parameters:
- `q` (inventory): 0 for symmetric book
- `gamma` (risk aversion): configurable, higher = wider spreads
- `sigma` (volatility): from core's vol field
- `T - t` (time remaining): derived from match clock
- `k` (order arrival intensity): calibrated constant

**5-level book generation:**
- Best bid/ask from `r - delta/2` and `r + delta/2`
- 4 additional levels at fixed tick increments (e.g., 0.25)
- Size per level: decaying from best (e.g., 100, 200, 300, 200, 150)

### publisher.py
WebSocket fan-out — the outbound side of the replica.

- Tracks set of connected WebSocket clients
- `connect(ws)` / `disconnect(ws)` — manage client set
- `async broadcast(world_state)` — serialize to JSON, send to all connected clients
- Per-client error handling: drop dead connections silently

## Lifecycle

```
startup:
  FastAPI boots via lifespan context manager
  async Redis client connects (redis.asyncio)
  consumer task created via asyncio.create_task()
  consumer starts in IDLE (is_active=False)
  WS endpoint ready, accepting connections

activation (POST /control {"action": "wake"}):
  last_id = "$"         # <-- skip history, only new events from now
  is_active = True
  consumer begins await r.xread() loop

steady state (10Hz):
  await r.xread() returns new event
  parse -> engine.update() -> await publisher.broadcast()
  track lag_ms for observability

chaos mode (POST /chaos {"lag_ms": 5000}):
  injected_lag = 5000ms
  each broadcast does await asyncio.sleep(lag/1000) first
  replica falls behind, lag_ms climbs
  Redis retains events — no data loss, just delayed delivery

deactivation (POST /control {"action": "sleep"}):
  is_active = False
  consumer returns to idle sleep loop
  WS connections stay open but stop receiving updates

shutdown:
  lifespan context manager cancels consumer task
  Redis connection closed
```

## Tests (pytest + httpx / FastAPI TestClient)

Run: `pytest replica/tests/ -v`

### test_avellaneda_stoikov_spread_logic
Pass a mock event with low volatility, compute the spread. Pass another with high
volatility. Assert the high-vol spread is strictly wider.

```python
from replica.engine import ReplicaEngine

engine = ReplicaEngine()

low_vol_event = {"ball_x": 50, "ball_y": 25, "vol": 0.1, "mid": 70.0,
                 "score": "0-0", "half": 1, "clock": 2700, "ts": 0}
engine.update(low_vol_event)
low_spread = engine.book.asks[0][0] - engine.book.bids[0][0]

high_vol_event = {**low_vol_event, "vol": 0.45}
engine.update(high_vol_event)
high_spread = engine.book.asks[0][0] - engine.book.bids[0][0]

assert high_spread > low_spread
```

### test_orderbook_monotonicity
Generate a 5-level book. Assert bids are descending, asks are ascending,
and no crossed book (best bid < best ask).

```python
engine = ReplicaEngine()
engine.update({"ball_x": 50, "ball_y": 25, "vol": 0.2, "mid": 70.0,
               "score": "0-0", "half": 1, "clock": 2700, "ts": 0})

bids = engine.book.bids  # [[price, size], ...]
asks = engine.book.asks

# Bids descending
for i in range(len(bids) - 1):
    assert bids[i][0] > bids[i + 1][0]

# Asks ascending
for i in range(len(asks) - 1):
    assert asks[i][0] < asks[i + 1][0]

# No crossed book
assert bids[0][0] < asks[0][0]
```

### test_state_update_convergence
