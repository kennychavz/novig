# 3.1 Core Implementation Plan

## Context

Read before starting:
- `docs/2.1_core.md` — full spec (directory structure, module responsibilities, data contract, tests)
- `docs/1_high_level.md` — system overview, canonical payloads, Redis Streams transport

Key design decision: The core has **no network server** (no FastAPI, no WebSockets). It is a
pure producer — tick loop + Redis XADD. Therefore use **synchronous Python** throughout:
standard `redis.Redis` client, `time.sleep()`, no asyncio. Simpler, no event loop overhead.

## Step 1: Config + Engine (game logic) ✅ DONE

Create `core/config.py` and `core/engine.py`.

**config.py**: All constants — tick rate, field dimensions, goal zone, pricing params,
volatility params, match duration, Redis URL, stream key, maxlen. Straight from 2.1.

**engine.py**: The `Engine` class with all game logic:
- `__init__`: ball at (50, 25), velocity (0, 0), score "0-0", half 1, clock 0
- `tick(dt)` method that:
  1. Perturbs velocity (random walk with momentum)
  2. Updates position: `x += vx * dt`, `y += vy * dt`
  3. Clamps to field boundaries, bounces off walls (y=0, y=50)
  4. Checks goal detection (x<=0 or x>=100, within y=20..30)
  5. On goal: increments score, resets ball to center
  6. Advances clock by dt
  7. Checks halftime (clock>=2700) and full time (clock>=5400)
  8. Computes mid-price: `base_price + (x / 100) * price_range`
  9. Computes volatility: base_vol with spike near goals, smooth transition
  10. Returns flat dict with all 8 fields (ts, ball_x, ball_y, vol, mid, score, half, clock)

**Commit**: `feat(core): add config and game engine`

**Implementation Notes:**
- Position update uses `x += vx` (velocity per-tick) instead of `x += vx * dt` — makes vx=5 move 5 units/tick, matching test expectations for goal detection.
- tick() builds the event dict BEFORE perturbing velocity for next tick, so setting ball_x and calling tick() returns the expected mid/vol for that position.
- Velocity perturbation: `gauss(0, 0.5)` per tick with 0.95 damping factor for smooth ball movement.

## Step 2: Publisher + Main (transport + entry point) ✅ DONE

Create `core/publisher.py`, `core/main.py`, and `core/requirements.txt`.

**publisher.py**: `Publisher` class using **synchronous** `redis.Redis`:
- `__init__(redis_url)`: `self.r = redis.Redis.from_url(redis_url, decode_responses=True)`
- `publish(event_dict)`: `self.r.xadd(STREAM_KEY, event_dict, maxlen=STREAM_MAXLEN, approximate=True)`
- Flattens all values to strings for Redis
- Reconnect with exponential backoff on connection loss

**main.py**: Entry point — **synchronous loop, no asyncio**:
```python
import time, signal
from engine import Engine
from publisher import Publisher
import config

running = True
signal.signal(signal.SIGINT, lambda *_: globals().update(running=False))
signal.signal(signal.SIGTERM, lambda *_: globals().update(running=False))

engine = Engine()
pub = Publisher(config.REDIS_URL)
dt = 1.0 / config.TICK_RATE_HZ

while running:
    event = engine.tick(dt)
    pub.publish(event)
    time.sleep(dt)
```

No async, no event loop. The core is a single-threaded producer — this is the correct
architecture since it serves no network requests.

**requirements.txt**: `redis>=5.0.0`

**Commit**: `feat(core): add sync redis publisher and main entry point`

**Implementation Notes:**
- main.py uses `time.monotonic()` for accurate tick timing — sleeps for the remainder of the tick interval after processing.
- Added `engine.finished` check in the main loop to stop when match ends.

## Step 3: Tests + Dockerfile ✅ DONE

Create `core/tests/test_engine.py` and `core/Dockerfile`.

**test_engine.py**: All 5 tests from 2.1:
- `test_physics_boundaries` — 1000 ticks, assert ball stays in bounds
- `test_goal_detection_and_reset` — force ball near goal, assert score + reset
- `test_pricing_extremes` — x=0, x=50, x=100 -> expected mid prices
- `test_volatility_spikes` — midfield vs danger zone vol values
- `test_redis_payload_serialization` — flat dict, all 8 keys, no nested types

**Dockerfile**:
```dockerfile

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["python", "main.py"]
```

Run tests: `pytest core/tests/ -v`

**Commit**: `feat(core): add tests and Dockerfile`

**Implementation Notes:**
- Tests zero `ball_vx`/`ball_vy` before setting position to prevent physics perturbation from prior ticks affecting results.
- Pricing/vol tests set `ball_y=10` (outside goal zone 20-30) when testing at x=0 or x=100 to avoid triggering goal detection.
- Volatility test uses `round(..., 4)` on expected value to match engine's rounding and avoid float precision mismatch (0.15*3.0 = 0.44999... in IEEE 754).
